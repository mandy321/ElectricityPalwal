# Security Policy

We take the security of this tracker and its users seriously. Since this project handles utilities data and interacts with public APIs, this document explains our security model and how to report vulnerabilities.

---

## 🔒 Security Model & Architecture

This application operates as a static Jamstack web app combined with automated scrapers.

### 1. Public Data Only
- All electricity outage and grid restoration data displayed on the dashboard is public information scraped directly from the official Dakshin Haryana Bijli Vitran Nigam (DHBVN) portal.
- **No private data** (such as customer account names, emails, telephone numbers, or payment info) is queried, scraped, or stored.

### 2. Client-Side CORS Proxy (`corsproxy.io`)
- To enable real-time updates when a user selects a district or clicks "Refresh Live", the frontend browser makes XML/SOAP requests directly to the DHBVN backend APIs.
- Because the DHBVN server does not send Cross-Origin Resource Sharing (CORS) headers, the app routes these requests via `https://corsproxy.io/`.
- No sensitive user metadata or custom auth tokens are sent through the proxy. Only the base64-encoded XML payload specifying the district ID and public static header tokens is transmitted.

### 3. No API Keys Exposed
- The DHBVN Samadhan portal relies on static HTTP header credentials generated by their Appsavy web framework. 
- These keys are public and static for anonymous outage forms. No private API keys, cloud provider credentials, or user secrets are embedded in the client-side code.

---

## 🐛 Reporting a Vulnerability

If you discover a security vulnerability or bug (such as exposure of internal paths, crawler manipulation, or workflow security risks):

1. **Do not** open a public GitHub issue.
2. Please report the issue privately by emailing the repository owner at the contact email listed on their GitHub profile.
3. Provide a clear description of the vulnerability, steps to reproduce it, and any potential impact.

We will review your report and release a fix as soon as possible.
